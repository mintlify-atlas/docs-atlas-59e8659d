This guide covers common problems you may encounter with RewindOS and how to resolve them.

## Daemon Not Starting

### Check Service Status

First, check if the daemon is running:

```bash
systemctl --user status rewindos-daemon
```

If it shows `inactive (dead)` or `failed`, check the logs:

```bash
journalctl --user -u rewindos-daemon -n 50
```

### Common Causes

<Accordion title="Daemon binary not found">

**Error:** `Failed to start rewindos-daemon.service: Exec format error`

**Solution:**

```bash
# Verify binary exists
which rewindos-daemon

# If missing, reinstall
make install-daemon
```

The daemon should be at `~/.local/bin/rewindos-daemon`.

</Accordion>

<Accordion title="D-Bus connection failed">

**Error:** `Failed to connect to D-Bus session bus`

**Solution:**

Check if D-Bus session bus is running:

```bash
echo $DBUS_SESSION_BUS_ADDRESS
```

If empty, D-Bus isn't available. This is rare on modern systems.

Restart your desktop session:

```bash
loginctl terminate-user $USER
```

Or reboot.

</Accordion>

<Accordion title="PipeWire not running">

**Error:** `PipeWire connection failed` or `Failed to start capture backend`

**Solution:**

Check PipeWire status:

```bash
systemctl --user status pipewire
```

If inactive, start it:

```bash
systemctl --user start pipewire
```

Install PipeWire if missing:

```bash
sudo apt install pipewire pipewire-pulse
```

<Note>
RewindOS requires PipeWire for screen capture on Wayland. X11 systems may use different backends.
</Note>

</Accordion>

<Accordion title="xdg-desktop-portal not installed">

**Error:** `xdg-desktop-portal not found`

**Solution:**

```bash
# Install portal and backend for your DE
sudo apt install xdg-desktop-portal xdg-desktop-portal-kde  # KDE Plasma
# OR
sudo apt install xdg-desktop-portal xdg-desktop-portal-gnome  # GNOME
```

Restart the daemon:

```bash
systemctl --user restart rewindos-daemon
```

</Accordion>

<Accordion title="Database corruption">

**Error:** `Failed to open database` or `database disk image is malformed`

**Solution:**

1. Stop the daemon:

```bash
systemctl --user stop rewindos-daemon
```

2. Backup and remove corrupted database:

```bash
cd ~/.rewindos
cp rewindos.db rewindos.db.backup
rm rewindos.db
```

3. Restart daemon (will create fresh DB):

```bash
systemctl --user start rewindos-daemon
```

<Warning>
You will lose search history, but screenshots remain in `~/.rewindos/screenshots/`.
</Warning>

</Accordion>

## OCR Failures

### Tesseract Not Found

**Symptom:** Logs show `tesseract not found — OCR will be disabled`

**Solution:**

Install Tesseract:

```bash
sudo apt install tesseract-ocr tesseract-ocr-eng
```

Verify installation:

```bash
tesseract --version
```

Restart daemon:

```bash
systemctl --user restart rewindos-daemon
```

### OCR Language Not Installed

**Symptom:** OCR works but misses non-English text

**Solution:**

Install additional language packs:

```bash
# German
sudo apt install tesseract-ocr-deu

# French
sudo apt install tesseract-ocr-fra

# Spanish
sudo apt install tesseract-ocr-spa

# Chinese (Simplified)
sudo apt install tesseract-ocr-chi-sim
```

Update config (`~/.rewindos/config.toml`):

```toml
[ocr]
enabled = true
tesseract_lang = "eng+deu"  # Multiple languages separated by +
max_workers = 2
```

Restart daemon.

### OCR Queue Backing Up

**Symptom:** `frames_ocr_pending` keeps growing in status output

**Solution:**

Increase OCR workers in config:

```toml
[ocr]
max_workers = 4  # Increase from 2 (default)
```

Or reduce capture frequency:

```toml
[capture]
interval_seconds = 10  # Increase from 5
```

Restart daemon.

<Info>
OCR is CPU-intensive. Balance workers vs. system performance. Recommended: 2-4 workers.
</Info>

## Ollama Connection Issues

### Ollama Not Reachable

**Symptom:** Logs show `Ollama not reachable at http://localhost:11434`

**Solution:**

<Steps>

<Step title="Check if Ollama is running">

```bash
systemctl status ollama
# OR
curl http://localhost:11434
```

Expected response: `Ollama is running`

</Step>

<Step title="Start Ollama if stopped">

```bash
# If installed as systemd service
sudo systemctl start ollama

# OR run manually
ollama serve
```

</Step>

<Step title="Verify RewindOS config">

Check `~/.rewindos/config.toml`:

```toml
[semantic]
enabled = false  # Auto-detected, but can force disable
ollama_url = "http://localhost:11434"
model = "nomic-embed-text"

[chat]
enabled = true
ollama_url = "http://localhost:11434"
model = "qwen2.5:7b"
```

Update URL if Ollama runs on a different host/port.

</Step>

<Step title="Restart daemon">

```bash
systemctl --user restart rewindos-daemon
```

Check logs:

```bash
journalctl --user -u rewindos-daemon | grep -i ollama
```

You should see: `Ollama detected — semantic search enabled`

</Step>

</Steps>

### Embedding Model Not Found

**Symptom:** `embedding model not found, pulling...` followed by timeout

**Solution:**

Manually pull the model:

```bash
ollama pull nomic-embed-text
```

Verify:

```bash
ollama list
```

You should see `nomic-embed-text` in the list.

### Chat Model Missing

**Symptom:** Ask (AI chat) returns errors

**Solution:**

Pull the chat model:

```bash
ollama pull qwen2.5:7b
```

Or use a different model in config:

```toml
[chat]
model = "llama3:8b"  # Alternative
```

<Tip>
Smaller models like `qwen2.5:3b` work well for chat and use less RAM. Recommended: 3b-7b models.
</Tip>

### Semantic Search Not Working

**Symptom:** Search always returns `"search_mode": "keyword"`

**Checklist:**

1. Ollama is running and reachable
2. `nomic-embed-text` model is pulled
3. Daemon detected Ollama on startup (check logs)
4. Existing screenshots have embeddings (run `rewindos-daemon backfill`)

```bash
# Verify embeddings exist
sqlite3 ~/.rewindos/rewindos.db "SELECT COUNT(*) FROM embeddings;"
```

If count is 0, run:

```bash
rewindos-daemon backfill
```

## Disk Space Issues

### Disk Usage Growing Too Fast

**Symptom:** `~/.rewindos/` directory is very large

Check current usage:

```bash
du -sh ~/.rewindos/
du -sh ~/.rewindos/screenshots/
```

### Solutions

<Accordion title="Reduce retention period">

Edit `~/.rewindos/config.toml`:

```toml
[storage]
retention_days = 30  # Reduce from 90
```

Restart daemon. Old screenshots will be deleted automatically.

</Accordion>

<Accordion title="Increase capture interval">

```toml
[capture]
interval_seconds = 10  # Increase from 5
```

Reduces screenshots/day from ~17k to ~8.5k.

</Accordion>

<Accordion title="Recompress existing screenshots">

Reduce image quality and dimensions:

```bash
rewindos-daemon recompress --quality 70 --max-width 1600 --dry-run
```

Review space savings, then run without `--dry-run`:

```bash
rewindos-daemon recompress --quality 70 --max-width 1600
```

<Warning>
Recompression is destructive. Screenshots will be overwritten with lower quality versions.
</Warning>

</Accordion>

<Accordion title="Manually delete old data">

Delete screenshots before a specific date:

1. Stop daemon:

```bash
systemctl --user stop rewindos-daemon
```

2. Remove old directories:

```bash
cd ~/.rewindos/screenshots/
rm -rf 2025-01-*  # Delete all January 2025 screenshots
```

3. Clean up database (optional):

```bash
sqlite3 ~/.rewindos/rewindos.db "DELETE FROM screenshots WHERE timestamp < strftime('%s', '2025-02-01');"
sqlite3 ~/.rewindos/rewindos.db "VACUUM;"
```

4. Restart daemon:

```bash
systemctl --user start rewindos-daemon
```

</Accordion>

### Storage Recommendations

At default settings (5s interval, 80 quality, 1920px width):

| Duration | Disk Usage |
|----------|------------|
| 1 day | ~210 MB |
| 1 week | ~1.5 GB |
| 1 month | ~6.3 GB |
| 90 days (default) | ~19 GB |

<Info>
Actual usage varies based on screen resolution, content complexity, and deduplication rate.
</Info>

## Capture Issues

### No Screenshots Being Captured

**Checklist:**

1. Daemon is running:

```bash
systemctl --user status rewindos-daemon
```

2. Capture is not paused:

```bash
rewindos-daemon status | grep is_capturing
```

Should show `"is_capturing": true`.

If paused, resume:

```bash
rewindos-daemon resume
```

3. Check logs for errors:

```bash
journalctl --user -u rewindos-daemon -f
```

4. Verify screenshots directory:

```bash
ls -la ~/.rewindos/screenshots/$(date +%Y-%m-%d)/
```

Should contain `.webp` files with recent timestamps.

### Screen Capture Permission Denied

**Symptom:** Logs show permission errors or portal dialogs keep appearing

**Solution:**

<Steps>

<Step title="Grant persistent permissions">

**KDE Plasma:**

The KWin ScreenShot2 method requires a desktop file. Verify it exists:

```bash
ls ~/.local/share/applications/com.rewindos.Daemon.desktop
```

Reinstall if missing:

```bash
make install-daemon
```

**GNOME:**

Grant permission in:
**Settings → Privacy → Screen Recording → RewindOS → Enable**

</Step>

<Step title="Update desktop database">

```bash
update-desktop-database ~/.local/share/applications/
```

</Step>

<Step title="Restart daemon">

```bash
systemctl --user restart rewindos-daemon
```

</Step>

</Steps>

### Deduplication Too Aggressive

**Symptom:** Important screenshots are being skipped

**Solution:**

Lower the change detection threshold in config:

```toml
[capture]
change_threshold = 1  # Reduce from 3 (default)
```

Lower values = more sensitive to changes = fewer frames deduplicated.

Restart daemon.

<Note>
Threshold is the hamming distance between perceptual hashes. Range: 0-64. Default: 3.
</Note>

## UI Issues

### Global Hotkey Not Working

See the [Keyboard Shortcuts troubleshooting section](/guides/keyboard-shortcuts#troubleshooting).

### App Doesn't Start

**Solution:**

Run from terminal to see errors:

```bash
rewindos
```

Common issues:

1. **Binary not found:**

```bash
make build
make install
```

2. **WebView2 missing (rare on Linux):**

```bash
sudo apt install webkit2gtk-4.1
```

3. **Config file corrupted:**

```bash
mv ~/.rewindos/config.toml ~/.rewindos/config.toml.backup
rewindos  # Creates fresh config
```

### Search Returns No Results

**Checklist:**

1. Daemon is running and has captured screenshots
2. OCR is enabled and working (check logs)
3. Try a simpler query (single word)
4. Check database:

```bash
sqlite3 ~/.rewindos/rewindos.db "SELECT COUNT(*) FROM screenshots;"
sqlite3 ~/.rewindos/rewindos.db "SELECT COUNT(*) FROM fts_screenshots;"
```

Both should return counts > 0.

5. Rebuild FTS index (if corrupted):

```bash
sqlite3 ~/.rewindos/rewindos.db "INSERT INTO fts_screenshots(fts_screenshots) VALUES('rebuild');"
```

## Performance Issues

### High CPU Usage

**Common causes:**

1. **Too many OCR workers:**

```toml
[ocr]
max_workers = 2  # Reduce from 4
```

2. **Capture interval too short:**

```toml
[capture]
interval_seconds = 10  # Increase from 5
```

3. **Ollama embedding backfill running:**

Check logs for "background embedding backfill". This is temporary and will complete.

### High Memory Usage

**Solution:**

1. Restart daemon (releases accumulated memory):

```bash
systemctl --user restart rewindos-daemon
```

2. Use a smaller Ollama model:

```toml
[chat]
model = "qwen2.5:3b"  # Smaller than 7b
```

3. Reduce max context tokens:

```toml
[chat]
max_context_tokens = 4096  # Reduce from 6144
```

## Getting Help

If your issue isn't covered here:

1. **Check logs:**

```bash
journalctl --user -u rewindos-daemon -n 200 > daemon-logs.txt
rewindos 2>&1 | tee ui-logs.txt
```

2. **Collect system info:**

```bash
uname -a
echo $XDG_SESSION_TYPE
echo $XDG_CURRENT_DESKTOP
tesseract --version
ollama --version
```

3. **Open a GitHub issue:**

[https://github.com/jay/rewindos/issues](https://github.com/jay/rewindos/issues)

Include:
- System info
- Relevant log excerpts
- Steps to reproduce
- Config file (redact any sensitive paths)
