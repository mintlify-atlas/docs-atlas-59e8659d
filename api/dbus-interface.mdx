The RewindOS daemon exposes a D-Bus service on the **session bus** for control by the Tauri UI, CLI tools, and desktop integration.

## Service Details

| Field | Value |
|---|---|
| Bus | Session (user) |
| Bus Name | `com.rewindos.Daemon` |
| Object Path | `/com/rewindos/Daemon` |
| Interface | `com.rewindos.Daemon` |

### Search

Search OCR text across all screenshots using FTS5 full-text search or hybrid search (FTS5 + vector similarity).

<ParamField path="query" type="string" required>
  FTS5 search query. Supports:
  - Boolean operators: `AND`, `OR`, `NOT`
  - Phrases: `"exact phrase"`
  - Prefix matching: `term*`
</ParamField>

<ParamField path="filters_json" type="string" required>
  JSON-encoded filter object. Pass `"{}"` for default filters.
</ParamField>

**Filter Schema:**

<CodeGroup>
```json Filter Example
{
  "start_time": 1706054400,
  "end_time": 1706140800,
  "app_name": "firefox",
  "limit": 50,
  "offset": 0
}
```
</CodeGroup>

<Expandable title="Filter Fields (all optional)">
  <ParamField path="start_time" type="i64">
    Filter results after this Unix timestamp (inclusive)
  </ParamField>
  
  <ParamField path="end_time" type="i64">
    Filter results before this Unix timestamp (inclusive)
  </ParamField>
  
  <ParamField path="app_name" type="string">
    Filter by application name (partial match)
  </ParamField>
  
  <ParamField path="limit" type="i64" default="50">
    Maximum number of results to return
  </ParamField>
  
  <ParamField path="offset" type="i64" default="0">
    Number of results to skip (for pagination)
  </ParamField>
</Expandable>

<ResponseField name="results_json" type="string">
  JSON-encoded search response with results and metadata
</ResponseField>

**Response Format:**

<CodeGroup>
```json Response Schema
{
  "results": [
    {
      "id": 12345,
      "timestamp": 1706137200,
      "app_name": "firefox",
      "window_title": "Stack Overflow - How to...",
      "thumbnail_path": "screenshots/2025-01-25/thumbs/1706137200.webp",
      "file_path": "screenshots/2025-01-25/1706137200.webp",
      "matched_text": "...the <mark>PostgreSQL</mark> connection pool was...",
      "rank": -12.5,
      "group_count": 5,
      "group_screenshot_ids": [12345, 12340, 12335, 12330, 12325]
    }
  ],
  "total_count": 142,
  "search_mode": "hybrid"
}
```
</CodeGroup>

<Expandable title="Response Fields">
  <ResponseField name="results" type="array" required>
    Array of search result objects
    <Expandable title="Result Object">
      <ResponseField name="id" type="i64" required>
        Screenshot ID
      </ResponseField>
      <ResponseField name="timestamp" type="i64" required>
        Unix timestamp when captured
      </ResponseField>
      <ResponseField name="app_name" type="string | null">
        Application name
      </ResponseField>
      <ResponseField name="window_title" type="string | null">
        Window title at capture time
      </ResponseField>
      <ResponseField name="thumbnail_path" type="string | null">
        Path to thumbnail image
      </ResponseField>
      <ResponseField name="file_path" type="string" required>
        Path to full screenshot
      </ResponseField>
      <ResponseField name="matched_text" type="string">
        OCR text snippet with `<mark>` tags around matches
      </ResponseField>
      <ResponseField name="rank" type="f64">
        FTS5 rank score (negative = better match)
      </ResponseField>
      <ResponseField name="group_count" type="i64">
        Number of visually similar screenshots grouped under this result (only present when > 1)
      </ResponseField>
      <ResponseField name="group_screenshot_ids" type="array[i64]">
        IDs of all screenshots in the scene deduplication group (only present when grouped)
      </ResponseField>
    </Expandable>
  </ResponseField>
  
  <ResponseField name="total_count" type="i64" required>
    Total number of matches (before pagination)
  </ResponseField>
  
  <ResponseField name="search_mode" type="string" required>
    Search strategy used:
    - `"keyword"` — FTS5 full-text search only
    - `"hybrid"` — FTS5 + vector similarity fused with Reciprocal Rank Fusion (RRF)
  </ResponseField>
</Expandable>

## Rust Client Interface

The Tauri UI connects to the daemon using `zbus::Connection`:

```rust
let dbus = zbus::blocking::Connection::session()
    .map(|c| c.into_inner())?;

let reply = dbus
    .call_method(
        Some("com.rewindos.Daemon"),
        "/com/rewindos/Daemon",
        Some("com.rewindos.Daemon"),
        "GetStatus",
        &(),
    )
    .await?;

let status_json: String = reply.body().deserialize()?;
```
